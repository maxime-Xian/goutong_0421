# 复杂问题处理方法论

> 融合 ECA v2.2 + v6.6 增强版

---

## 阶段0：智能分诊

**复杂问题判定**（满足任一即为复杂）：

| 特征 | 检测信号 |
|------|----------|
| 资源分配 | 涉及时间/金钱/注意力分配 |
| 多个未知变量 | 存在不确定因素 |
| 结果在未来验证 | 决策效果需时间检验 |
| 用户犹豫 | 明显在权衡 |

**分诊规则**：
- 简单问题 → 直接回答（模式A）
- 复杂问题 → 走methodology流程（模式B）
- 状态问题 → 降复杂度+不推动决策（模式C）

---

## 阶段1：问题界定

**必做动作**：

1. **用自己的话重述**
   ```
   "我理解你的问题是..."
   ```

2. **明确目标**
   ```
   "你希望达成的结果是？"
   ```

3. **检查框架陷阱**
   - "这个问题本身是否假设了某个框架？"
   - "有没有 alternative definition？"

---

## 阶段2：信息收集

**必做动作**：

1. **扫描 knowledge/**
   ```
   grep -l "关键词" $KNOWLEDGE_DIR/*.md
   ```

2. **检查 case_index**
   - 是否有相似案例？
   - 上次如何解决？

3. **补充事实**
   - 必要的事实层信息有哪些？
   - 哪些是推测？

---

## 阶段3：方案生成

**必做动作**：

1. **至少3个选项**
   - 方案A（最优假设）
   - 方案B（保守方案）
   - 方案C（否决/放弃方案）

2. **每个方案标注**
   - 优点：
   - 缺点：
   - 置信度：？（高/中/低）

3. **包含否决方案**
   - "这个问题可能不需要解决"
   - "保持现状可能是最优解"

---

## 阶段4：决策输出

**输出格式**：

```
## 结论
[一句话概括]

## 分析
【事实】已知确定的信息
【推测】高置信度的推断
【假设】低置信度的假设，需要验证

## 推荐
首选方案：[方案X]
理由：[简短理由]

## 风险
[主要风险点]
```

---

## 阶段5：复盘触发

**必做动作**：

1. **记录到 memory/guardian/reflections/**
   ```
   YYYY-MM-DD_complex_issue.md
   ```

2. **标记待验证项**
   - 需要在何时验证结果？
   - 验证指标是什么？

3. **如果是重大决策**
   - 同时记录到 episodes/

---

## 与 ECA 联动

| 阶段 | ECA层级 | 产出 |
|------|--------|------|
| 阶段0 | 信号门控 | 问题分类 |
| 阶段1 | 熵控制层 | 问题界定 |
| 阶段2 | 信号门控 | 信息筛选 |
| 阶段3 | 战略推理 | 方案生成 |
| 阶段4 | 战略推理 | 决策输出 |
| 阶段5 | 进化层 | 闭环记录 |

---

## 禁止事项

- ❌ 不在阶段1停留超过10分钟
- ❌ 不跳过阶段2直接生成方案
- ❌ 不只给1个选项
- ❌ 不在无事实支撑时使用"确定""必然"
