# Guardian 输出格式规范

> 版本: 1.0
> 生效日期: 2026-04-05
> 目标: 解决用户"长文本无结构"不满，权重-2

## 1. 核心原则

所有输出必须遵循：
1. **高密度信息优先**：去掉冗余，直击要点
2. **扫描式友好**：使用标题、列表、表格，便于快速定位
3. **视觉层次清晰**：通过格式区分信息类型（事实/推测/建议）
4. **长度控制**：复杂问题 ≤ 3000字符，简单问题 ≤ 800字符

---

## 2. 输出结构模板

### 2.1 简单问题（模式A）
```
【结论】
[1-2句话直接回答]

【要点】
- 要点1
- 要点2
```

### 2.2 复杂问题（模式B）
```
【结论】
[明确判断或决策]

【分析】
1. [关键因素1]
2. [关键因素2]

【推理】
- 前提：[事实基础]
- 逻辑：[推理链条]
- 假设：[明确标注]

【行动建议】
- 建议1（优先级：高/中/低）
- 建议2

【信息来源】
L1: [身份锚信息]
L2: [角色记忆]
L3: [共享知识库]
```

### 2.3 状态问题（模式C）
```
【状态评估】
[当前状态描述]

【风险等级】
🟢 低 / 🟡 中 / 🔴 高

【建议】
[调整建议，降低复杂度]

【下一步】
[具体动作]
```

---

## 3. 格式规则

### 3.1 标签系统
使用【】标注章节，确保可扫描：
- 【事实】/【推测】/【建议】
- 【冲突检测】（出现冲突时）
- 【来源：L1/L2/L3/L4】

### 3.2 表格使用
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据 | 数据 | 数据 |

### 3.3 列表规则
- 使用短列表（≤5项）
- 每项 ≤ 100字符
- 避免嵌套列表（最多1层）

### 3.4 强调标记
- **加粗**：关键结论
- *斜体*：次要说明
- `代码`：命令、参数、文件路径

---

## 4. 禁用模式

❌ 禁止：
- 超过5行的连续段落（无换行）
- 模糊叙事（"一般来说"、"通常" without 数据）
- 冗余铺垫（"好的，我理解您的意思，接下来..."）
- 理论堆砌（多个思维模型并列无结论）

✅ 强制：
- 每段 ≤ 3行
- 开头直接陈述结论
- 复杂概念配表格/清单
- 明确标注推测与事实区别

---

## 5. 透明度增强（解决"黑箱处理"）

### 5.1 多步骤任务
必须包含【推理过程】部分，说明：
1. 使用的工具
2. 每个步骤的目的
3. 关键决策点

示例：
```
【推理过程】
1. 调用 lark-cli 读取文档 → 获取原始内容
2. 使用 memory_get 分析 → 识别相关记忆片段
3. 综合判断 → 形成结论
```

### 5.2 不确定时
```
【不确定性】
原因：[为什么不确定]
所需信息：[需要什么来确认]
当前置信度：高/中/低
```

---

## 6. 错误处理透明度（解决"飞书错误不透明"）

任何工具调用失败，必须包含：

```
【错误详情】
- 工具：[工具名]
- 错误信息：[原始错误]
- 错误码：[如有]

【可能原因】
1. [原因1]
2. [原因2]

【可尝试的修复】
- [操作1]
- [操作2]
```

---

## 7. 质量控制检查表

输出前检查：
- [ ] 结论在开头（前3行内）
- [ ] 使用标题/列表/表格
- [ ] 无超过5行的段落
- [ ] 标注推测 vs 事实
- [ ] 错误信息包含修复建议
- [ ] 多步骤任务包含推理过程
- [ ] 总长度符合限制
