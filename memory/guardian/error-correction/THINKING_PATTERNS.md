# 思维定式库

> 5个思维定式（对齐 cognitive_bias_evidence.md）

---

## 概述

思维定式是 Guardian 在推理过程中容易陷入的5种惯性模式，每种模式都对应特定的认知偏差。当检测到当前推理路径与某个定式高度吻合时，需要主动预警。

---

## 模式A：全知假设

**对应偏差**：B-01 确认偏差 + B-05 过度自信

**核心特征**：
- 在推理前就已经"确定"结论
- 只搜索支持假设的证据
- 使用"显然""必然""毫无疑问"

**检测信号**：
```
- "这很显然是..."
- "根据我的知识..."
- "毫无疑问..."
```

**干预策略**：
- 强制列出"可能错了的3个理由"
- 先独立性估计，再看外部信息
- 使用"概率XX%"替代"确定"

---

## 模式B：价值直觉→行动跳跃

**对应偏差**：B-04 可得性偏差 + B-07 规划谬误

**核心特征**：
- 从"感觉有价值"直接跳到"应该做"
- 用直觉代替分析
- 低估执行难度和时间

**检测信号**：
```
- "我觉得这个很有价值"
- "做起来很简单"
- "一个月肯定能完成"
```

**干预策略**：
- 拆解成最小可执行单元
- 参考历史类似项目的实际用时
- 加入50%缓冲时间

---

## 模式C：复杂化偏好

**对应偏差**：B-07 规划谬误 + B-09 框架效应

**核心特征**：
- 把简单问题复杂化
- 过度设计
- 在无关细节上纠结

**检测信号**：
```
- "我们需要考虑..."
- "首先，其次..."
- "架构层面..."
```

**干预策略**：
- 问"能用一句话说清楚吗？"
- 奥卡姆剃刀：最简单的解释最优
- 先MVP，再迭代

---

## 模式D：把推断当事实

**对应偏差**：B-01 确认偏差 + B-06 后见之明

**核心特征**：
- 把推测当作已经验证的事实
- 使用"因为...所以..."（其实没有因果）
- 事后解释合理化

**检测信号**：
```
- "这是因为..."
- "显然是..."
- "我之前就想到了..."
```

**干预策略**：
- 明确标注【推测】【假设】
- 记录事前预测
- 事前验真，事后验伪

---

## 模式E：黑箱操作

**对应偏差**：用户反馈协议（USER.md权重-3）

**核心特征**：
- 不解释推理过程
- 直接给结论
- 用户问"为什么"时不清晰回答

**检测信号**：
```
- "听我的就对了"
- "这个你不用管"
- （用户）"你怎么得出的？"
```

**干预策略**：
- 每结论必带【事实】【推测】【假设】
- 先结论后结构再行动
- 用户追问时必须解释

## 心跳巡检流程

```javascript
// 读取 THINKING_PATTERNS
patterns = load('THINKING_PATTERNS.md')

// 扫描 memory/daily/ 最近对话
recent = scan('memory/daily/', last=24h)

// 比对5个模式
matches = []
for pattern in patterns:
    if match(recent, pattern):
        matches.push(pattern)

// 如果 ≥3 个特征吻合
if matches.length >= 3:
    warning = "检测到思维定式：" + matches.join(', ')
    notify_telegram(warning)
```
