# 主动学习闭环协议

> v1.0 — 2026-03-31

---

## 核心原则

**错误不积累，闭环不过夜，同一错误不出现第二次。**

---

## 错误分类

| 类型 | 定义 | 处理方式 |
|------|------|----------|
| **类型A** | Guardian犯错（用户指出"你错了"） | 实时记录→分析→修正 |
| **类型B** | Maxime犯错（用户说"我之前..."） | 被动记录→确认→更新知识库 |
| **类型C** | 外部事件（cron失败/系统异常） | 系统级修复→通知 |

---

## 闭环流程

```
错误发生
    ↓
记录到 error-correction/[TYPE]/[DATE]_[ID].md
    ↓
用思维模型分析根因（cognitive_bias联动）
    ↓
发Telegram告知 Maxime（类型A实时，类型B在晚间）
    ↓
更新 case_index（如果是有价值的案例）
    ↓
更新 THINKING_PATTERNS（如果触发思维定式）
```

---

## 错误记录模板

```markdown
# 错误记录：[类型]

## 基本信息
- 时间：YYYY-MM-DD HH:mm
- 来源：用户反馈 / 系统日志
- 类别：类型A / 类型B / 类型C

## 错误描述
[具体内容]

## 根因分析
关联思维模型：[模型名称]
分析过程：[分析]

## 处理结果
[如何修复]

## 经验教训
[防止再次发生的规则]
```

## 主动提醒触发

| 触发条件 | 通知时机 | 内容 |
|----------|----------|------|
| 类型A错误 | 实时 | "我发现了一个错误：[简述]" |
| 类型B历史重犯 | 每15分钟心跳 | "你之前提到过XX，现在又..." |
| 类型C系统异常 | 实时 | "系统异常：[描述]+修复建议 |

---

## 心跳巡检

**每15分钟检查**（在 heartbeat 服务中）：

```javascript
// 1. 扫描 error-correction/ 类型A
// 2. 对比 THINKING_PATTERNS 的5个模式
// 3. 如果 ≥3 个特征吻合 → 预警
// 4. 发 Telegram 提醒
```

---

## 与其他框架联动

| 框架 | 联动点 |
|------|--------|
| cognitive_bias | 根因分析，思维模型选择 |
| THINKING_PATTERNS | 思维定式检测，模式匹配 |
| MEMORY | 长期存储，冲突解决 |
| AGENTS | 如果错误因推理流程问题，修正AGENTS规则 |
