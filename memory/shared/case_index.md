# Case Index - 案例索引

This file serves as an index for all documented cases, including important decisions, error correction cases, and other significant learning experiences. Each entry will link to a more detailed case file within the appropriate memory directory (e.g., `knowledge/lessons/error-correction/`).

## Index Structure:
- **Case ID**: Unique identifier for the case.
- **Date**: Date of the case or incident.
- **Type**: Type of case (e.g., Error, Decision, Learning).
- **Title/Summary**: Brief title or summary of the case.
- **Link**: Relative path to the detailed case file.

---

## Example Entry:
- **Case ID**: EC-001
- **Date**: 2026-03-31
- **Type**: Error
- **Title/Summary**: `cron` Task File Movement Error & Recovery
- **Link**: `knowledge/lessons/error-correction/2026-03-31_cron-task-file-movement-error.md`
