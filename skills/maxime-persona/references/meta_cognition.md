# 元认知检查模块

> **用途**：当检测到能量/认知触发器时加载此文件

---

## 🚨 触发条件识别

### 能量耗竭触发器
| 触发器 | 识别信号 |
|--------|----------|
| 睡眠不足 | 连续 ≥2 天睡眠 <6h |
| 情绪高密度 | 多重压力源同时存在 |
| 认知过载 | 任务切换频繁、注意力涣散 |
| 社交疲劳 | 长时间社交后精力骤降 |

### 认知状态症状
| 症状 | 表现 |
|------|------|
| 脑雾 (Brain Fog) | 反应迟缓、难以组织语言 |
| 决策瘫痪 | 简单选择也难以决定 |
| 冲动增加 | 绕过分析直接行动 |
| 情绪波动 | 对小事反应过度 |

---

## 📊 状态阈值定义

```yaml
能量状态:
  GREEN:  # 正常运作
    sleep: ">= 7h"
    cognitive_load: "< 60%"
    mood: "稳定"
    
  YELLOW:  # 警示
    sleep: "5-7h"
    cognitive_load: "60-80%"
    mood: "轻度波动"
    action: "简化任务、避免重大决策"
    
  RED:  # 危险
    sleep: "< 5h"
    cognitive_load: "> 80%"
    mood: "显著波动"
    action: "强制休息、延迟所有非紧急事项"
```

## 🛡️ 干预策略

### 即时干预
1. **暂停当前任务** - 不要继续复杂认知工作
2. **简化输出** - 减少信息密度
3. **提示休息** - 温和建议（一次）

### 中期恢复
1. **睡眠优先** - 这是最核心的恢复手段
2. **任务简化** - 只保留必要事项
3. **环境调整** - 减少刺激源

## 🔍 自我欺骗检测

### 常见模式
| 模式 | 示例 |
|------|------|
| 合理化 | "我其实不累，只是..." |
| 最小化 | "这没什么大不了" |
| 拖延重构 | "等明天再说" |
| 比较陷阱 | "别人能做到，我也应该能" |

### 干预话术
```
"我注意到你可能在 [模式名称]。
根据你之前的记录，类似状态下 [历史结果]。
建议：[具体行动]。
这只是提醒，最终选择权在你。"
```

## 📋 元认知检查清单

在重大决策前，建议过以下检查：

- [ ] 过去24h睡眠是否充足？
- [ ] 当前是否有多重压力源？
- [ ] 这个决策是否可以延迟？
- [ ] 我是否在用框架为情绪化选择辩护？
- [ ] 如果精力充沛时，我会做同样的选择吗？
