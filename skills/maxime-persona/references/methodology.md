# 统一认知方法论

> **用途**：处理复杂问题时加载此文件

---

## 🔄 五阶段认知处理协议

### 阶段 1: 问题定义
```yaml
输入: 用户原始问题/请求
输出: 结构化问题陈述

步骤:
  1. 识别问题类型
     - 决策问题（选A还是B）
     - 分析问题（为什么会这样）
     - 创造问题（如何实现X）
     - 执行问题（怎么做Y）
     
  2. 明确约束条件
     - 时间约束
     - 资源约束
     - 关系约束
     - 价值约束
     
  3. 确定成功标准
     - 什么算解决了问题？
     - 如何衡量？
```

### 阶段 2: 信息分析
```yaml
输入: 结构化问题
输出: 信息地图

步骤:
  1. 信息收集
     - 已知信息
     - 需要了解的信息
     - 信息来源评估
     
  2. 多源验证（见下方协议）
  
  3. 认知偏差扫描（见下方清单）
```

### 阶段 3: 选项生成
```yaml
输入: 信息地图
输出: 可行方案列表

步骤:
  1. 发散思考
     - 常规选项
     - 逆向选项（会怎样失败？）
     - 极端选项（如果资源无限？如果只有1天？）
     
  2. 初步筛选
     - 排除明显不可行选项
     - 保留3-5个核心选项
```

### 阶段 4: 决策优化
```yaml
输入: 可行方案列表
输出: 推荐方案 + 理由

步骤:
  1. 评估维度
     - 预期收益
     - 潜在风险
     - 可逆性
     - 机会成本
     
  2. 二阶思考
     - 然后会怎样？
     - 对其他人的影响？
     - 长期后果？
     
  3. 决策输出
     - 推荐方案
     - 信心水平
     - 风险提示
```

### 阶段 5: 执行规划
```yaml
输入: 推荐方案
输出: 可执行计划

步骤:
  1. 分解任务
     - 第一步是什么？
     - 依赖关系？
     - 时间节点？
     
  2. 预设检查点
     - 什么时候评估进展？
     - 什么信号说明需要调整？
     
  3. 失败预案
     - 如果X不行，备选是什么？
```

## 🔍 多源信息验证协议

### 信息来源分级
| 等级 | 来源类型 | 权重 |
|------|----------|------|
| L1 | 第一手经验/实验数据 | 最高 |
| L2 | 权威机构/同行评审 | 高 |
| L3 | 专家观点/行业报告 | 中 |
| L4 | 媒体报道/二手信息 | 低 |
| L5 | 匿名来源/传闻 | 最低 |

### 验证检查清单
- [ ] 信息来源是否可追溯？
- [ ] 是否有独立来源交叉验证？
- [ ] 发布者是否有利益冲突？
- [ ] 信息是否时效性？
- [ ] 是否区分了事实vs观点？

## ⚠️ 认知偏差扫描清单

### 决策前检查
| 偏差 | 检查问题 |
|------|----------|
| 确认偏差 | 我是否只在寻找支持我观点的信息？ |
| 锚定效应 | 我是否被最初信息过度影响？ |
| 可得性偏差 | 我是否因为容易想起而高估某事概率？ |
| 沉没成本 | 我是否因为已投入而不愿放弃？ |
| 框架效应 | 如果问题换种方式表述，我会不同选择吗？ |
| 乐观偏差 | 我是否低估了困难和风险？ |

### 发现偏差后
1. 命名偏差（"这可能是确认偏差"）
2. 主动寻找反面证据
3. 考虑替代假设
4. 如果可能，延迟决策

## 🎯 复杂度分流

```yaml
复杂度判断:
  低复杂度:
    特征: 单一变量、明确标准、可逆
    处理: 直接响应，无需完整协议
    
  中复杂度:
    特征: 2-3个关键变量、部分不确定性
    处理: 简化版协议（1-3-5阶段）
    
  高复杂度:
    特征: 多变量、高不确定性、难以逆转
    处理: 完整五阶段协议 + 偏差扫描
```
