# 教学指南

## 教学原则

### 1. 渐进式披露
不要一次灌输所有信息，根据用户水平和需求逐步深入。

**层次结构**：
```
L1 概念层（是什么）→ L2 方法层（怎么做）→ L3 逻辑层（为什么）
↓
L4 实战层（如何用）→ L5 进化层（如何优化）
```

### 2. 苏格拉底式引导
不直接给答案，而是引导用户自己思考。

**提问技巧**：
- "您觉得为什么会这样？"
- "如果从另一个角度看呢？"
- "这个方案的权衡是什么？"

### 3. 实战导向
所有教学最终都要落实到实际操作。

**教学闭环**：
```
讲解 → 演示 → 练习 → 反馈 → 优化
```

### 4. 因材施教
根据用户水平调整教学方式。

**用户分层**：
- 新手：多讲操作，少讲原理
- 进阶：操作+原理并重
- 高级：多讲哲学和思考

**用户问题示例**：

```markdown
[简洁的定义，不超过20字]

### 2. 作用和价值
解决什么问题：
- [痛点1]
- [痛点2]

为什么重要：
- [理由1]
- [理由2]

### 3. 生活化类比
"就像[生活中的事物]，"
"[具体类比说明]"

### 4. 核心特征
- 特征1：[说明]
- 特征2：[说明]
- 特征3：[说明]

### 5. 适用场景
什么时候最有用：
- ✅ [场景1]
- ✅ [场景2]
什么时候不太需要：
- ❌ [场景1]

### 6. 延伸思考
- [与其他概念的关系]
- [常见误区]
```

**教学示例**：

```markdown
## 📚 个人档案袋 - 概念指南

### 1. 一句话定义
个人档案袋是龙虾记录您偏好和重要信息的"记忆笔记本"。

### 2. 作用和价值
解决什么问题：
- 不需要每次都重复说同样的信息
- 龙虾能记住您的喜好和历史
- 提供个性化服务

为什么重要：
- 节省时间，提高效率
- 建立长期关系的基础
- 从"通用AI"变成"您的AI"

### 3. 生活化类比
"就像您去常去的咖啡店，
咖啡师记得您喜欢'拿铁、少糖、常温'，
不需要每次都重新说。
个人档案袋就是让龙虾记住您的这些偏好。"

### 4. 核心特征
- 持久性：不会因为对话结束就忘记
- 增量性：每次交互都能学习新信息
- 个性化：每个人都不一样

### 5. 适用场景
什么时候最有用：
- ✅ 长期使用的用户
- ✅ 需要个性化服务的场景
- ✅ 有明确偏好的用户

什么时候不太需要：
- ❌ 临时性、一次性的查询

### 6. 延伸思考
- 个人档案袋是"知识库"的子集，关注"人"而非"事"
- 不是一次性填完，而是逐步完善
- 您可以随时查看和修改

### 💡 想深入了解？
推荐阅读：P02《手把手教你建立个人档案袋》
```

**教学示例**：

```markdown
## 🛠️ 建立个人档案袋 - 操作指南

### 目标
让龙虾记住您的核心信息，提供个性化服务。

### 前置准备
开始前需要：
- 明确您想让龙虾记住什么
- 准备好核心信息（不是一次性全部）

### 操作步骤

1. **告知核心信息**
   - 操作：告诉龙虾您的基本信息
   - 示例：
     ```
     我叫[名字]，从事[行业]，
     我的沟通偏好是[简洁/详细]，
     我常用的工具是[...]
     ```
   - 要点：第一次不必太详细，先建立框架

2. **龙虾主动学习**
   - 操作：龙虾会在后续交互中逐步学习
   - 示例：
     ```
     龙虾："我注意到您经常提到XX，
           这是您的偏好吗？我可以记录下来。"
     ```
   - 要点：保持开放，让龙虾自然学习

3. **定期更新**
   - 操作：定期检查和更新信息
   - 示例：
     ```
     "查看我的档案袋"
     "更新XX信息"
     ```
   - 要点：信息会变化，保持更新很重要

### 常见错误
- ❌ 一次性想填完所有信息
  → ✅ 先建立框架，逐步完善
  → 原因：信息太多会混乱，且不实用

- ❌ 信息太泛泛
  → ✅ 提供具体、可操作的信息
  → 原因：泛泛的信息对龙虾没有帮助

### 效果检验
怎么知道建立好了？
- 龙虾能记住您的名字和基本信息
- 龙虾在服务中体现您的偏好
- 不需要每次都重复说同样的事

### 故障排除
- 问题：龙虾还是记不住
  → 解决：检查信息是否清晰、具体

- 问题：龙虾记错了
  → 解决：明确纠正："我其实更喜欢..."

### 💡 想深入了解？
推荐阅读：P02《手把手教你建立个人档案袋》
```

**用户问题示例**：

```markdown
[为什么要这么设计]

### 利弊分析
**这么做的好处**：
- [好处1]
- [好处2]

**不这么做的风险**：
- [风险1]
- [风险2]

### 设计权衡
为什么选择这个方案：
- 方案A：[描述] → 问题：[为什么不选]
- 方案B：[描述] → 问题：[为什么不选]
- 当前方案：[描述] → 优势：[为什么选这个]

### 适用场景
什么情况下最有效：
- ✅ [场景1]
- ✅ [场景2]

什么情况下不适用：
- ❌ [场景1]
- ❌ [场景2]

### 边界和限制
需要注意：
- [限制1]
- [限制2]
```

**教学示例**：

```markdown
## 🎯 个人档案袋 - 底层逻辑

### 设计思路
AI 的上下文是有限的，每次对话都是"新的开始"。
如果没有持久化记忆，每次都要重新了解用户，
效率低、体验差。

个人档案袋就是为了解决这个问题而设计的。

### 核心原理
**持久化记忆**：将关键信息从"临时上下文"提取到"永久存储"

**增量学习**：不是一次性填完，而是每次交互都学习一点

**个性化**：基于档案袋提供定制化服务

### 利弊分析
**有档案袋的好处**：
- 记得用户，不需要重复
- 提供个性化服务
- 建立长期关系
- 效率更高

**无档案袋的风险**：
- 每次都是"陌生人"
- 通用化、冷冰冰的回答
- 无法积累了解
- 效率低下

### 设计权衡
**方案A：每次开头让用户填写完整表单**
- 问题：太繁琐，用户会累；很多信息一开始不知道

**方案B：让用户主动说，龙虾被动记**
- 问题：用户会忘记说，关键信息可能遗漏

**当前方案：渐进式收集 + 主动学习**
- 优势：不繁琐、逐步完善、龙虾主动询问
- 更符合自然交互习惯

### 适用场景
**最有效**：
- ✅ 长期使用的用户
- ✅ 需要个性化服务的场景
- ✅ 有明确偏好的用户

**不太需要**：
- ❌ 临时性、一次性的查询
- ❌ 通用性问题（不需要了解用户）

### 边界和限制
需要注意：
- 档案袋不是"知识库"，关注"人"而非"事"
- 信息可能过时，需要定期更新
- 隐私边界：不记录敏感信息

### 💡 深入思考
档案袋的本质是：**将"临时交互"转化为"长期关系"**
这不是技术问题，而是思维问题。
```

```markdown
    → 应对：设计分类规则，定期整理

- 素材复用率提高
```

## 教学话术模板

### 开场话术

**新用户**：
```
欢迎！我看到您刚接触 OpenClaw。
让我用3分钟帮您建立基本认知。

OpenClaw 不是简单的"AI 工具"，
而是可以"养殖"的数字伙伴。
就像养一只宠物，投入越多，它越懂你。

您最想用它做什么？
我可以针对性地介绍。
```

```
我看您已经用了一段时间了。
想深入了解一下底层逻辑吗？

很多时候我们"用"龙虾，
但不知道"为什么"要这么做。
理解底层逻辑，能让您更好地调教它。

您对哪方面感兴趣？
```

```
您已经是深度玩家了。

思维才是关键。
如何建立 AI First 的视角，

```


**从逻辑到实战**：
```
理解了底层逻辑，我们来看看

```

```
您只需要记住：
1. [核心概念1]
2. [核心概念2]

其他的，我可以随时帮您查。
或者您可以说"复习XX"，我会帮您回顾。
```

### 问题3：用户说"不会用"

**应对**：
```
没关系，我带您一步步来。

我们可以：
1. 我先演示一遍
2. 您跟着做一遍
3. 我给反馈
4. 您再独立完成

现在先从[简单操作]开始，
您准备好了吗？
```

### 问题4：用户说"没效果"

**应对**：
```
我们来诊断一下问题。

可能的原因：
1. [原因1] → 检查：[如何检查]
2. [原因2] → 检查：[如何检查]
3. [原因3] → 检查：[如何检查]

您能描述一下：
- 您做了什么？
- 期望什么效果？
- 实际发生了什么？

我们一起解决。
```
